# Taxi Fare MLOps Pipeline

## Описание пайплайна

В данном проекте реализутся  MLOps pipeline для задачи предсказания стоимости поездки на такси.

Пайплайн состоит из **Airflow ML** и **ClearML**, развёрнутых на предоставленной ВМ. **Clear ML Agent** задеплоен в **kubernetis** с помощью **helm**:
- **Airflow DAG** — обработка и подготовка данных
    + `airflow/final_task_dag.py` для Airflow переносится в папку ./dag на ВМ вручную.
- **ClearML Pipeline** — обучение, оценка и выбор лучшей модели
    + `clearml/pipeline.py` подтягивается с гита после локального запуска "uv run airflow/pipeline"


---

## Шаги пайплайна

### Airflow (ETL)

DAG выполняется каждый час и включает:

1. **Проверка наличия данных в S3**
2. **Загрузка данных из S3**
3. **Валидация через Pandera**
4. **Предобработка:**
   - удаление пропусков
   - фильтрация выбросов
   - генерация признаков (`hour`, `day_of_week`, `distance`)
5. **Сохранение обработанных данных обратно в S3**

---

### ClearML Pipeline

Pipeline состоит из следующих шагов:

1. **load_data**
   - загрузка подготовленного датасета из S3
   - разбиение на train/test

2. **train_models (параллельно)**
   - обучение нескольких моделей RandomForest:
     - small
     - medium
     - large

3. **evaluate**
   - расчёт метрики RMSE для каждой модели

4. **select_best_model**
   - выбор модели с минимальным RMSE
   - сохранение модели в ClearML и хранилище S3

---

## Гиперпараметры

Гиперпараметры задаются в шаге обучения моделей:

```python
n_estimators = 50 / 100 / 200
max_depth = 8 / 10 / 12
```
